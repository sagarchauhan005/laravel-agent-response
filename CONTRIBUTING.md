# Contributing

Thank you for considering contributing to Laravel llms.txt Package!

## Code Style

This package follows PSR-12 coding standards and uses Laravel Pint for code formatting.

```bash
composer pint
```

## Testing

Run the test suite:

```bash
composer test
```

## Pull Requests

1. Fork the repository
2. Create your feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Run tests and Pint
5. Commit your changes (`git commit -m 'Add some amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## Reporting Issues

Please use the GitHub issue tracker to report bugs or suggest features.

## Suggesting Presets

If you'd like to suggest a new use-case preset, please open an issue with:
- Use case name
- Description
- Suggested sections
- Example use case
